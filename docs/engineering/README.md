# Engineering Docs

This directory is for implementation-facing documentation that should stay separate from product strategy and roadmap files.

## Purpose
- Capture the current technical architecture.
- Record engineering decisions that affect implementation order.
- Keep future setup, data model, and system notes close to the codebase without mixing them into product planning documents.

## Current Files
- `architecture.md` outlines the current frontend-only baseline and the recommended next technical layers.

## Suggested Future Additions
- `backend-architecture.md` for service and real-time session design.
- `data-model.md` for rounds, guesses, sessions, teams, and leaderboard entities.
- `deployment.md` for hosting, storage, environments, and release steps.
- `testing-strategy.md` for unit, integration, and end-to-end validation plans.
